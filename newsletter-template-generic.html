🤖 AI Corner

Integrá el módulo de pagos en menos de 30 segundos con IA

La Biblioteca incluye prompts prediseñados para los casos de uso más comunes. Acá un ejemplo de cómo generar una integración lista para entorno de pruebas:

Prompt de ejemplo
Claude / ChatGPT / Gemini
// Copiá este prompt desde la Biblioteca de Recursos "Actuá como experto en integración de la plataforma Acme. Generame el código Node.js completo para crear una transacción con: - 1 ítem de $5.000 - URLs de retorno configuradas para sandbox - referencia externa única con UUID - manejo de errores con try/catch Usá el SDK oficial y variables de entorno."
Resultado: El modelo genera el snippet con el SDK, UUID, variables de entorno y manejo de errores — listo para pegar en tu proyecto. Sin asumir endpoints no documentados, sin hardcodear credenciales.

💡 Tip: Si tenés el MCP Server conectado en tu IDE, los prompts de la biblioteca funcionan en modo guiado — el modelo accede a la documentación oficial en tiempo real mientras genera el código.

💻 Code Snippet

Guardá un método de pago y procesá la segunda transacción sin pedir datos de nuevo

Flujo completo de tokenización con la nueva sección unificada. Funciona en sandbox con tus credenciales actuales.

save-payment-method.js
Node.js
const AcmeSDK = require('acme-sdk'); const client = new AcmeSDK({ accessToken: process.env.ACME_ACCESS_TOKEN }); // 1. Crear o recuperar cliente const customer = await client.customers.create({ email: 'dev@ejemplo.com', }); // 2. Asociar método de pago tokenizado al cliente const paymentMethod = await client.paymentMethods.create({ customerId: customer.id, token: req.body.paymentToken, // token generado en el frontend }); // 3. Segunda transacción: usar método guardado const payment = await client.payments.create({ transaction_amount: 5000, token: paymentMethod.id, // sin pedir datos al usuario installments: 1, payer: { id: customer.id }, });

Validado en sandbox. El paymentToken se genera con el SDK en el frontend — nunca viajan los datos sensibles a tu servidor.

📋 Changelog — Top del mes

Los cambios más relevantes de Febrero–Marzo

  • ⚠ Acción requerida
    Ofuscamiento de datos en endpoint transactions: datos del pagador y del método de pago se enmascaran en transacciones no aprobadas. Revisá qué campos usás actualmente y capturá los datos del comprador en tu propio sistema. Ver guía de migración
  • ✦ Nuevo
    API de Órdenes — disponible en nuevos países: nueva API que consolida múltiples soluciones de pago en una sola interfaz. Errores completos en una sola respuesta, webhooks configurables desde el dashboard. Ver docs

  • ✦ Nuevo
